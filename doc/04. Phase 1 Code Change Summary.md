### Critical patterns applied
- **`@Modifying(clearAutomatically = true)`** on all bulk `UPDATE`/`DELETE` queries â€” prevents stale cache where a subsequent `find` could return a cached entity that was deleted/revoked.
- **`@PrePersist` / `@PreUpdate`** lifecycle callbacks on entities with `created_at` / `updated_at` columns.
- **Named parameter queries** with `@Param` annotations in all `TokenRepository` and `AuthCodeRepository` bulk operations.
- **Comma-separated TEXT** for array columns (`redirect_uris`, `scopes`, `grant_types`, `groups`, `members`, `emails`, `phones`) â€” no JSON columns except `attributes` (JSONB on SCIM entities).

### Bugs fixed during Task 3
- **Module `relativePath`** â€” all module POMs had `relativePath=..` (pointed to `backend/pom.xml` instead of root `pom.xml`). Fixed to `relativePath=../../pom.xml` across all 8 modules.

## Phase 1 Review Fixes (`ea79f13`)

### Critical issues fixed

**1. `TotpCredential.updated_at` NOT NULL constraint violation**
- **Problem:** `totp_credential.updated_at TIMESTAMPTZ NOT NULL` in `V1__init.sql`, but `TotpCredential.java` entity had no `updatedAt` field or `@PreUpdate` callback. Any JPA `persist()` or `merge()` would throw `SQLException: NULL value in column "updated_at"`.
- **Fix:** Added `updatedAt` field with `@Column(name = "updated_at", nullable = false)` and `@PreUpdate protected void onUpdate()` lifecycle callback.

**2. `@Modifying` missing `clearAutomatically = true` â€” stale cache bug**
- **Problem:** `TokenRepository.revokeByJti()`, `TokenRepository.revokeAllBySubject()`, `TokenRepository.deleteExpired()`, `AuthCodeRepository.markConsumed()`, `AuthCodeRepository.deleteExpired()`, `DeviceCodeRepository.deleteExpired()` all had `@Modifying` but were missing `clearAutomatically = true`. After a bulk UPDATE/DELETE, the persistence context was not cleared, so subsequent `findByJtiAndRevokedFalse()` could return a stale cached entity with `revoked = false`.
- **Fix:** Added `clearAutomatically = true` to all 6 bulk operations across 3 repositories.

## Architecture Decisions

| Decision | Rationale |
|----------|-----------|
| Flyway in `auth-core` only | JPA entities and schema live in `auth-core`; all other modules depend on it |
| Redis is short-lived state store; PostgreSQL is source of truth | Auth codes (5min TTL), device codes (10min TTL), refresh tokens (7d TTL) stored in Redis; clients, tokens, users in PostgreSQL |
| `JPA @Query` for bulk UPDATE/DELETE with `@Modifying` | Standard Spring Data JPA pattern; `clearAutomatically = true` is critical for cache consistency |
| H2 in-memory DB for unit tests | Fast, deterministic, no external dependency required |
| UUID as primary key for `Token`, SCIM entities, MFA, directory | PostgreSQL native UUID generation via `gen_random_uuid()` |
| String primary key for `OAuthClient`, `AuthCode`, `DeviceCode` | Opaque tokens generated externally or as secure random strings, not auto-generated |
| JSONB only for SCIM `attributes` field | SCIM spec requires flexible attributes; all other "array" columns use comma-separated TEXT |
